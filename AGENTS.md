# Stock Survey Agent

这是一个专业的股票调查 agent 项目。所有工作都应围绕股票研究、资料整理、调查结论沉淀和可追溯的数据归档展开。

## 调查结果存放约定

- 所有股票调查结果统一放在项目根目录的 `data/` 文件夹下。
- 每只股票单独使用一个以股票代码命名的子文件夹存放资料，例如：
  - `data/AAPL/`
  - `data/TSLA/`
  - `data/600519/`
- 同一只股票的原始资料、整理笔记、分析报告、引用来源和中间产物都应保存在该股票代码对应的文件夹内。
- 不同股票之间的资料不要混放，避免后续检索、复盘和自动化处理时出现歧义。

## 调查工作要求

- 优先保证资料来源清晰、结论可追溯。
- 保存调查结果时，应尽量记录来源、日期、关键假设和重要风险点。
- 对尚未验证的信息要明确标注，不要把推测写成确定结论。
- 涉及最新股价、财报、公告、监管动态或新闻时，应以最新可验证来源为准。
- 获取 A 股财报 PDF 时，优先使用本项目的 `.agents/skills/fetch-eastmoney-reports` skill；东方财富可作为自动下载镜像，正式研究结论仍需回到交易所、巨潮资讯网或公司官方披露交叉核对。

## PDF 读取约定

- 读取财报 PDF 正文和表格时，优先使用 `pdfplumber`。
- 当前系统 `python3` 不一定安装了 `pdfplumber`；Codex 工作区 bundled runtime 中可用。
- 如需读取 PDF，先通过 Codex 工作区依赖信息获取 bundled Python 路径，不要把本机绝对路径写入仓库文件。
- 不要只依赖 `pypdf` 判断文本可读性；部分中文财报在 `pypdf` 下可能出现乱码，而 `pdfplumber` 可正常提取。

## 安全与提交约束

- 不要把本机绝对路径、用户名、缓存目录、临时目录或机器特有配置写入仓库文件。
- 不要提交任何 token、API key、密码、私钥、cookie、会话信息或其他敏感凭证。
- 不要提交 `.tmp/`、缓存、日志、虚拟环境、依赖目录或一次性调试产物。
- push 前必须执行安全扫描，至少检查本机路径和常见敏感字段模式。
- push 前必须检查 staged diff，确认只包含本次任务需要提交的内容。
- 发现疑似敏感信息时，先移除并重新扫描，不要带着不确定内容继续提交。
